# 고객 니즈 분석 프롬프트

다음 통화에서 고객의 니즈와 요구사항을 분석하세요.

## 고객 발화
{{customer_text}}

## 전체 대화
{{conversation}}

## 분석 항목

### 1. 주요 사유
- 고객이 전화한 가장 큰 이유 (한 문장)

### 2. 구체적 요구사항
- 고객이 원하는 것들 (배열)

### 3. 고민/문제점 (Pain Points)
- 고객이 해결하고 싶은 문제들

### 4. 긴급도
- 높음: 즉시 해결 필요
- 보통: 일반적인 문의
- 낮음: 정보 수집 단계

### 5. 구매 의향 신호
- 가격 질문, 결제 방법 문의, 시작 시기 질문 등

## 출력 형식

```json
{
  "primary_reason": "전화한 주요 사유 한 문장",
  "specific_needs": ["요구사항1", "요구사항2"],
  "pain_points": ["고민1", "고민2"],
  "urgency_level": "높음|보통|낮음",
  "buying_signals": ["가격 문의", "시작 시기 질문"],
  "objections": ["비용 부담", "기존 계약"]
}
```

JSON만 출력하세요.
